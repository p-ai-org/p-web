            <h3 class="h4 tPadding0 bPadding0 summary">
             <span class="expand_section">
              A calendaring tool is now available for W3C groups
             </span>
            </h3>
            <p class="date">
             <span class="dtstart published" title="2021-03-16T12:50:02Z">
              16 March 2021
             </span>
             |
             <a href="https://www.w3.org/blog/news/archives/8959" title="Archive: A calendaring tool is now available for W3C groups">
              Archive
             </a>
            </p>
           </div>
           <div class="description expand_description">
            <p>
             <a class="imageLink" href="https://www.w3.org/blog/2021/03/group-calendaring-service-enters-beta-test/">
              <img alt="screenshot of a detail of a group event" src="https://www.w3.org/blog/wp-content/uploads/2021/03/group_calendar-768x627.png" width="100"/>
             </a>
             Today we announced the
             <a href="https://www.w3.org/blog/2021/03/group-calendaring-service-enters-beta-test/">
              Beta of a group calendaring service
             </a>
             , developed by the
             <a href="https://www.w3.org/People/functions/systeam">
              W3C Systems Team
             </a>
             . The tool aims to help W3C Groups schedule their meetings and share them with the W3C community. A new “Calendar” tab is now available from our
             <a href="https://www.w3.org/groups/">
              group pages
             </a>
             , lists all upcoming meetings a given group has scheduled, and enables in particular viewing times in different time zones (including your own), exporting and subscribing to an iCal feed.
            </p>
            <p>
             This has been a long-standing request from W3C Groups to manage calendars of events, subscribe to them as well as easily view them on the Web, thus facilitating participation in, and scheduling of group meetings. You can read more in Jean-Gui Rouel’s blog posts about the tool’s
             <a href="https://www.w3.org/blog/2020/10/group-calendaring-discovery-phase/">
              discovery phase
             </a>
             and about
             <a href="https://www.w3.org/blog/2021/03/group-calendaring-service-enters-beta-test/">
              functionalities and future roadmap
             </a>
             .
            </p>

            <h3 class="h4 tPadding0 bPadding0 summary">
             <span class="expand_section">
              W3C Invites Implementations of Decentralized Identifiers (DIDs) v1.0
             </span>
            </h3>
            <p class="date">
             <span class="dtstart published" title="2021-03-18T09:19:03Z">
              18 March 2021
             </span>
             |
             <a href="https://www.w3.org/blog/news/archives/8966" title="Archive: W3C Invites Implementations of Decentralized Identifiers (DIDs) v1.0">
              Archive
             </a>
            </p>
           </div>
           <div class="description expand_description">
            <p>
             The
             <a href="https://www.w3.org/2019/did-wg/">
              Decentralized Identifier Working 
Group
             </a>
             invites implementations of a Candidate Recommendation of
             <a href="https://www.w3.org/TR/2021/CR-did-core-20210318/">
              Decentralized Identifiers (DIDs) v1.0
             </a>
             .
            </p>
            <p>
             This document defines Decentralized identifiers (DIDs), a new type of identifier that enables 
verifiable, decentralized digital identity. A DID identifies any subject (e.g., a 
person, organization, thing, data model, abstract entity, etc.) that the controller 
of the DID decides that it identifies. In contrast to typical, federated identifiers, 
DIDs have been designed so that they may be decoupled from centralized registries, 
identity providers, and certificate authorities.  DIDs are URIs that associate a DID 
subject with a DID document allowing trustable interactions associated with that 
subject. Each DID document can express cryptographic material, verification methods, 
or services, which provide a set of mechanisms enabling a DID controller to prove 
control of the DID.
            </p>
            <p>
             The Working Group has also published a separate
             <a href="https://www.w3.org/TR/did-use-cases/">
              Use Cases and Requirement document
             </a>
             that provides some background for the usage of this technology.
            </p>
            <p>
             Candidate Recommendation means that the Working Group considers the technical 
design to be complete, and is seeking implementation feedback on the document. The 
group is keen to get comments and implementation experiences on this specification as 
issues raised in the documents’
             <a href="https://github.com/w3c/did-core/">
              Github repository
             </a>
             .
            </p>
            <p>
             The group expects to satisfy the implementation goals (i.e., at least two, 
independent implementations for each of the test cases) by
             <strong>
              18 April 2021
             </strong>
             .
            </p>
           </div>

            <h3 class="h4 tPadding0 bPadding0 summary">
             <span class="expand_section">
              First Public Working Draft: Indexed Database API 3.0
             </span>
            </h3>
            <p class="date">
             <span class="dtstart published" title="2021-03-11T07:51:44Z">
              11 March 2021
             </span>
             |
             <a href="https://www.w3.org/blog/news/archives/8944" title="Archive: First Public Working Draft: Indexed Database API 3.0">
              Archive
             </a>
            </p>
           </div>
           <div class="description expand_description">
            <p>
             The
             <a href="https://www.w3.org/2019/webapps/">
              Web Applications Working Group
             </a>
             has published a First Public Working Draft of
             <a href="https://www.w3.org/TR/2021/WD-IndexedDB-3-20210311/">
              Indexed Database API 3.0
             </a>
             . This document defines APIs for a database of records holding simple values and hierarchical objects. Each record consists of a key and some value. Moreover, the database maintains indexes over records it stores. An application developer directly uses an API to locate records either by their key or by using an index. A query language can be layered on this API. An indexed database can be implemented using a persistent B-tree data structure.
            </p>
            <p>
             This is the Third Edition of Indexed Database API, which is intended to supersede Indexed Database API 2.0.
            </p>

            <h3 class="h4 tPadding0 bPadding0 summary">
             <span class="expand_section">
              W3C accepting proposals for Professional Employer Organization (PEO) Services
             </span>
            </h3>
            <p class="date">
             <span class="dtstart published" title="2021-03-04T10:34:31Z">
              4 March 2021
             </span>
             |
             <a href="https://www.w3.org/blog/news/archives/8935" title="Archive: W3C accepting proposals for Professional Employer Organization (PEO) Services">
              Archive
             </a>
            </p>
           </div>
           <div class="description expand_description">
            <p>
             <a href="https://www.w3.org/">
              <img alt="W3C" class="aligncenter size-medium" height="46" loading="lazy" src="https://www.w3.org/Icons/w3c_logo.svg" width="300"/>
             </a>
            </p>
            <p>
             Since its founding in 1994 by Web Inventor Tim Berners-Lee, the World Wide Web Consortium has developed the foundational technical standards upon which the Web has flourished. The Web and its place in society have changed dramatically, and the Web Consortium has been at the core of its technical interoperability. Today we need a more dramatic transformation to address the opportunities and threats the Web now faces and to continue to shape its future constructively.
            </p>
            <p>
             W3C is currently headquartered in Massachusetts, with staff members distributed around the world. Four institutions partner to “host” W3C: MIT (USA), ERCIM (France), Keio University (Japan), and Beihang University (China). We are planning the transition to a USA-based incorporated legal entity, a 501(c)(3) non-profit, as of January 1, 2022, which will continue to work with the distributed staff while consolidating financial and administrative functions. The Web Consortium is accepting proposals for Professional Employer Organization (PEO) Services; please consult our
             <a href="http://www.w3.org/2021/03/RFP-PEO-services.pdf">
              <abbr title="Request for Proposals">
               RFP
              </abbr>
             </a>
             .
            </p>
            <p>
             If your organization provides these services please submit a proposal by email to
             <a href="mailto:W3CLEadmin@w3.org">
              W3CLEadmin@w3.org
             </a>
             . If you have any questions, you can send them to that same address.
            </p>
            <p>
             Interested parties must respond
             <b>
              by Friday, April 2, 2021
             </b>
             , and we expect to make a selection by Friday, April 9, 2021. We look forward to getting some robust responses!
            </p>

            <h3 class="h4 tPadding0 bPadding0 summary">
             <span class="expand_section">
              WebRTC 1.0 is a W3C Recommendation
             </span>
            </h3>
            <p class="date">
             <span class="dtstart published" title="2021-01-26T14:10:45Z">
              26 January 2021
             </span>
             |
             <a href="https://www.w3.org/blog/news/archives/8897" title="Archive: WebRTC 1.0 is a W3C Recommendation">
              Archive
             </a>
            </p>
           </div>
           <div class="description expand_description">
            <p class="imageLink">
             <img alt="WebRTC" src="https://www.w3.org/comm/assets/logos/webrtc-logo-horiz-retro-750x140.png" width="240"/>
             The
             <a href="https://www.w3.org/groups/wg/webrtc">
              Web Real-Time Communications Working Group
             </a>
             has published
             <a href="https://www.w3.org/TR/2021/REC-webrtc-20210126/">
              WebRTC 1.0: Real-Time Communication Between Browsers
             </a>
             as a W3C Recommendation. This document defines a set of JavaScript APIs to allow media and generic application data to be exchanged with another browser or device implementing the appropriate set of real-time protocols defined in
             <a href="https://www.ietf.org/">
              IETF
             </a>
             .
            </p>
            <p>
             WebRTC already serves as a cornerstone of online communication and collaboration services. The WebRTC framework provides the building blocks from which web and app developers can seamlessly add video chat and peer-to-peer data exchange to a range of applications. Billions of users can interact now that WebRTC makes live video chat easier than ever on the Web. To learn more about this timely achievement and what the future holds for WebRTC, please, read our
             <a href="https://www.w3.org/2021/01/pressrelease-webrtc-rec.html.en">
              press release
             </a>
             .
            </p>
